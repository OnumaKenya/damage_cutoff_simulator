# ブルアカダメージ足切り最適化(β版) 利用マニュアル

## 概要

ブルーアーカイブの総力戦・大決戦等において、複数の攻撃カードによる**合計ダメージの分布**を計算し、目標ダメージの**通過確率**を算出するツールです。

ダメージのレンジ（会心・非会心の上下限、Hit数、会心率、回避率）は、**[Schale DB](https://schaledb.com/) のダメージ計算ツール**で得られた値をそのまま入力する想定です。Schale DB はダメージ減衰を考慮済みのため、後述の「減衰考慮済み」モードのまま使えます。スクリーンショットからの自動取り込み（OCR）も Schale DB の計算結果画面を対象としています。

このツールには2つのページがあります。

- **シミュレータ**: カードを並べて合計ダメージ分布と通過確率を計算します。
- **足切りライン最適化**: 同じ攻撃列を使い、複数チェックポイントでリセットする運用の**最適な足切りライン**を計算します（旧「足切りスライダー」機能の後継）。

## 2. サイドバー設定（シミュレータ）

### 計算方式

| 方式 | 説明 |
|---|---|
| **COS法（準厳密・推奨）** | 特性関数のフーリエ余弦級数反転＋原子分離(DP)で、分布・通過確率を**直接かつ準厳密**に計算します。乱数を使わないため結果が安定し、低確率（深い裾）の通過判定も信頼できます。 |
| **モンテカルロ** | 乱数サンプリングによる従来方式。比較・確認用に残しています。サンプル数が有限なため、低確率域はばらつきます。 |

通常は **COS法** のままで構いません。同じカード設定なら両者の分布・期待値はほぼ一致し、COS法の方が滑らかです。

### HP依存ダメージ

ミカ・フィーナのように、与ダメージが**敵の現在HPに比例する倍率**を持つキャラを扱うモードです。

| 設定 | 説明 |
|---|---|
| **なし（合計＝和モデル）** | 各ヒットのダメージを単純合計する通常モデル。 |
| **あり（ミカ型＝積モデル）** | 倍率が現在HPに線形依存し、累積ダメージが「独立な積」になるモデル。HP満タン時の倍率(R1)・HP0時の倍率(R0)・敵の最大HP・開始時HPを入力します。 |

「あり」を選ぶと入力欄が表示されます。倍率は現在HP $H_n$ に対し `倍率 = (R1−R0)/最大HP × H_n + R0` で線形に変化します（理論は `docs/product.md`）。足切りライン最適化でも積モデルに対応しています。

### ダメージ生成モード

| モード | 説明 |
|---|---|
| **減衰考慮済み（推奨）** | 入力するダメージ値は減衰適用後の値。Schale DB など減衰を考慮済みのツールの値をそのまま入力できます。 |
| **減衰考慮前** | 入力するダメージ値は減衰適用前の生ダメージ。内部で減衰関数が適用されます。 |

Schale DB の値を使う場合は「**減衰考慮済み**」のままにしてください。

### 安定値（全体設定）

> ※ かつてはカードごとの入力でしたが、**サイドバーの全体設定に移動**しました。全カードに共通で適用されます。

ダメージが大きいキャラでは、表示される最大ダメージが**減衰上限(10,966,999)で頭打ち**になり、本来の最大ダメージが分からなくなることがあります。このとき**安定値**を入力すると、信頼できる**最小ダメージ**から本来の最大ダメージを逆算します。

- 関係式: 減衰前の最小ダメージ ＝ 減衰前の最大ダメージ × `(1 − 1/(1 + 0.001 × 安定値) + 0.2)`
- これを解いて `減衰前の最大 ＝ 減衰前の最小 ÷ 比率` を求め、上限に張り付く分（上限到達分）を確率分布へ正しく反映します。
- **最大ダメージが上限未満なら安定値は無視され**、通常どおり入力値がそのまま使われます。未入力でも従来動作のままです。

## 3. 目標ダメージ・一括設定（カード一覧の上部）

### 目標ダメージ

達成したい合計ダメージを入力します。設定すると：
- 分布グラフに緑色の目標ラインが表示されます
- 目標を超える確率（通過確率）が計算されます

足切りライン最適化の目標ダメージ D は、解析ページ側で別途入力します。

### 一括設定

全カードの会心率・回避率をまとめて設定します。

1. 会心率(%)と回避率(%)を入力
2. 「**一括適用**」ボタンを押す → 全カードに反映

---

## 4. ダメージカード

各攻撃カード（EXスキル・NSなど）のダメージ情報を1枚ずつ入力します。

### 入力項目

| 項目 | 説明 |
|---|---|
| **会心ダメージ下限 / 上限** | 会心発生時のダメージ範囲 |
| **非会心ダメージ下限 / 上限** | 非会心時のダメージ範囲 |
| **Hit数** | そのカードの攻撃ヒット回数 |
| **会心率 (%)** | 会心が発生する確率（カード個別に設定可能） |
| **回避率 (%)** | 敵が回避する確率（カード個別に設定可能） |
| **敵の数** | Hit数に掛けて総ヒット数を算出します。全体攻撃などで複数体に同時に当たる場合に 2 以上を指定します（通常は 1）。 |

> ※ **安定値**はカードごとではなく、サイドバーの全体設定に移動しました（前述）。

### 操作

| 操作 | 方法 |
|---|---|
| カードの追加 | 「**+ ダメージ追加**」ボタン |
| カードの複製 | カード右上の「**📋**」ボタン（同じパラメータ・備考のカードが直後に追加されます） |
| カードの削除 | カード右上の「**✕**」ボタン（最低1枚は残ります） |
| カードの並べ替え | カード左上の「**⠿**」ハンドルをドラッグ＆ドロップ |
| メモの記入 | カード上部の備考欄に自由にテキストを入力 |

## 5. 入力の保存 / 読込（JSON）

カード一覧上部の「**💾 入力の保存 / 読込**」パネルで、現在の入力一式をファイルに保存・復元できます。

| 操作 | 方法 |
|---|---|
| エクスポート | 「**⬇ エクスポート (JSON)**」ボタン → 全カード・全体設定・足切りライン最適化の設定を JSON で保存 |
| インポート | 「**⬆ インポート**」エリアに JSON をドロップ、またはクリックして選択 |

- 保存される内容: 各カードのパラメータと備考、全体設定（会心率・回避率・計算方式・HP依存・ダメージ生成モード・安定値・目標ダメージ）、足切りライン最適化のチェックポイント／区間ごとの時間割合。
- インポートすると足切りライン最適化の足切り設定（チェックポイント・区間ごとの時間割合）も復元されます。

## 6. スクショからカード自動生成（OCR）

**Schale DB のダメージ計算ツール**のスクリーンショットから、カードを自動生成できます。カード一覧上部の「**📷 スクショからカード生成**」パネルを使います。

| 操作 | 方法 |
|---|---|
| 画像から取り込み | 画像ファイルをドロップ、またはクリックして選択 |
| 画面スニップ | 「**🖥 画面スニップ**」→ 取り込む画面/ウィンドウを選択 → ダメージ表示部分をドラッグで範囲選択 |

- ヒットごとの「通常 / 会心」のレンジ・単一値を自動判別してカード化します。
- **確定会心**（会心1行のみ）の表示は会心率100%として取り込みます。
- 同じダメージ値が連続するヒットは1枚のカードにまとめ、Hit数を合算します。
- **HP依存**の表示（現在HP等）を検出すると、サイドバーのHP依存モードを自動でONにします。
- 認識結果は必ず目視で確認してください。レイアウトによっては誤認識する場合があります。

> **設定**: この機能は Google Cloud Vision API を使用します。`GOOGLE_VISION_API_KEY` を環境変数または `.env`（`.env.example` 参照）に設定してください。

## 7. テキストからカード自動生成

ダメージ表示の**テキストをコピー＆ペースト**してカードを自動生成できます。OCR が使えない場合や、テキストを直接コピーできる場合に便利です。カード一覧上部の「**📝 テキストからカード生成**」パネルを使います。

| 操作 | 方法 |
|---|---|
| テキストから取り込み | テキスト入力欄にダメージ表示を貼り付け →「**テキストから取り込み**」ボタン |

ヒット行・ダメージ行・「会心」行が改行で分かれた、以下のような形式に対応します。

**一般（通常 + 会心）の例:**

```
ヒット1-2 (165.33%)
18,164 - 25,247
会心
35,239 - 48,979
ヒット3 (1322.61%)
145,315 - 201,977
会心
281,910 - 391,835
```

**確定会心 / 確定非会心（ダメージ1行のみ・「会心」行が続かない）の例:**

```
ヒット1-3 (415.83%)
24,893 - 31,980
ヒット4 (1524.72%)
91,276 - 117,259
```

- 「ヒットN-M」からヒット数を、続くダメージ行から通常レンジを、「会心」行の次のダメージ行から会心レンジを読み取ります。括弧内の倍率(%)は備考に記録されます。
- **「会心」行が続かない**ヒットは、OCR と同じく**確定会心**（会心率100%）として取り込みます。実際には**確定非会心**だった場合は、生成後にそのカードの会心率を `0` に修正してください。
- 同じダメージ値が連続するヒットは1枚のカードにまとめ、Hit数を合算します。
- **HP依存**の表示（現在HP等）を含むテキストを検出すると、サイドバーのHP依存モードを自動でONにします。
- 取り込み結果は必ず目視で確認してください。

## 8. シミュレーション実行

1. 各カードのダメージ情報を入力
2. 目標ダメージを設定（任意）
3. 「**シミュレーション実行**」ボタンを押す

### 結果の見方

- **分布グラフ**: COS法では確率密度の曲線、モンテカルロではヒストグラムで表示されます
- **それ以上になる確率グラフ（P(D≥x)）**: 横軸のダメージ x 以上になる確率（%）を表す曲線です。CDF を反転した値（1 − 累積確率）なので、目標ラインの高さがそのまま通過確率に対応します
- **赤い破線**: 期待値（平均ダメージ）
- **緑の実線**: 目標ダメージ（設定時のみ）。注記に通過確率が併記されます
- **通過確率**: 目標ダメージを超える確率がテキスト表示されます（末尾に COS法 / MC の別が付きます）
- **通過確率 ⇄ ダメージ 変換**: グラフの上・通過確率テキストの下にある変換パネルで、任意のダメージから通過確率（P(D≥x)）を、また任意の通過確率(%)から対応するダメージを、相互に計算できます。直近のシミュレーション結果（分布）に基づいて即座に更新されます。ダメージ欄はシミュレーション実行のたびに**目標ダメージ**が初期値として入力されます（その後は自由に書き換え可能で、次回実行時に再び目標ダメージへ戻ります）

## 9. 足切りライン最適化

複数のチェックポイントで**リセット（リスタ）する運用**において、各関門の**最適な足切りライン**を計算し、目標達成までのスループット（成功/時間）を最大化するページです。

各関門は「コスト/成功 = 期待時間/成功確率」を最小化する Bermudan 後ろ向き帰納で決定します（HP依存＝積モデルにも対応）。理論は `docs/cutoff.md`・`docs/restart_cos.md` を参照してください。

### 手順

1. **シミュレータ**ページで攻撃カードを設定する（足切りライン最適化はこの攻撃列を使います）。
2. ナビバーで「**足切りライン最適化**」に切り替える。
3. 「**カード読込 / 更新**」を押す（カードを変更したら都度押してください）。
4. 設定する：
   - **足切り（チェックポイント）の追加**: プルダウンから足切り判定を行う関門（カード）を選び「**+ 追加**」を押します（最終セグメントは候補に出ません）。
   - **区間カード**: 足切りで区切られた各区間が 1 枚のカードとして並びます（末尾は「完走」区間）。足切りを追加するとカードが分割され、各カードの「✕」でその足切りを解除できます。
   - **時間割合**: 各区間カードに、その区間に掛かる所要時間の相対値を入力します。コスト = 期待時間 / 成功確率 の計算に使われます。
5. **目標ダメージ D** を入力する。
6. 「**解析実行**」を押す。

### 結果の見方

- **図**: チェックポイント（カード）ごとの最適足切りライン（目標までの**残りダメージ**）。各点に区間/累積通過率を併記。残り0の「完走/目標達成」行が末尾に自動追加されます。
- **サマリ**: 成功率 / 平均所要時間 / スループット（成功/時間）。「足切り無し」との比較で**時短率（speedup）**を表示します。
- **表**:
  - **区間通過率** = そのチェックポイントに到達した試行のうち足切りを通過する割合（条件付き）。
  - **累積通過率** = 開始からそこまで全関門を通過する割合。最終行（完走/目標達成）の累積が目標達成率です。

### リスタラインを手で調整して確率の変化を見る

「解析実行」の結果の下に、**リスタライン手動調整**パネル（青枠）が表示されます。

- 各足切りの「**残りダメージ**」**スライダー**が、最適ラインの値で初期表示されます。
- スライダーを動かすと、その足切りラインでの**成功率・平均時間・スループット（時短率）**が即座に再計算され、グラフに反映されます。
- グラフは **青＝あなたの設定ライン** と **灰点線＝最適ライン** を重ねて表示します。各点には区間/累積通過率を併記。
- サマリに**最適比**（成功率の差・スループット比）を表示するので、最適からどれだけ離れているかが分かります。
- 「**最適ラインに戻す**」ボタンで全スライダーを最適値へ戻せます。

足切りを**厳しく**（残りダメージを小さく）すると通過率が下がり、**緩く**すると通過率は上がるが時間を浪費します。スループット（成功/時間）が最大になる点が最適ラインです。

### 残りダメージが途中で増加する場合

最適足切りライン（残りダメージ）が途中で**増加**することがあります。これは不具合ではなく、後続区間の所要時間（時間割合）が大きいほど、その手前の関門を厳しく（残りダメージを小さく）するのが最適なためです。関門を通過した後は累積ダメージが増えるだけなので、通過率がほぼ変わらない関門は実質的に拘束しておらず、設定上は無くても結果は変わりません。

## 10. 操作のヒント

- **カードの並び順**は実際の攻撃順に揃えてください。足切りライン最適化のチェックポイント位置は並び順（累積ヒット数）に依存します。
- 入力一式は **JSON で保存/読込**できます（セクション5）。
- 結果のグラフはPlotlyで描画されており、**ズーム・パン・ホバー**で詳細を確認できます。
- ダメージのレンジ・会心率などの元データは **[Schale DB](https://schaledb.com/)** のダメージ計算ツールで確認できます。
