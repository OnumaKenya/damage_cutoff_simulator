# ブルアカダメージ足切りシミュレータ 利用マニュアル

## 概要

ブルーアーカイブの総力戦・大決戦等において、複数の攻撃カードによる**合計ダメージの分布**をモンテカルロ法でシミュレーションし、目標ダメージの**通過確率**を算出するツールです。

さらに「**足切り**」機能により、カードの並び順で上側・下側にグループ分割し、足切りラインでのダメージ配分を視覚的に分析できます。

---

## 1. サイドバー設定

### ダメージ生成モード

| モード | 説明 |
|---|---|
| **減衰考慮済み（推奨）** | 入力するダメージ値は減衰適用後の値。ゲーム内で表示されるダメージをそのまま入力できます。 |
| **減衰考慮前** | 入力するダメージ値は減衰適用前の生ダメージ。内部で減衰関数が適用されます。 |

ダメージ減衰が考慮されている計算ツールを使う場合は、「**減衰考慮済み**」のまま使用してください。

### 目標ダメージ

達成したい合計ダメージを入力します。設定すると：
- ヒストグラムに緑色の目標ラインが表示されます
- 目標を超える確率（通過確率）が計算されます

### 一括設定

全カードの会心率・回避率をまとめて設定します。

1. 会心率(%)と回避率(%)を入力
2. 「**一括適用**」ボタンを押す → 全カードに反映

---

## 2. ダメージカード

各攻撃カード（EXスキル・NSなど）のダメージ情報を1枚ずつ入力します。

### 入力項目

| 項目 | 説明 |
|---|---|
| **会心ダメージ下限 / 上限** | 会心発生時のダメージ範囲 |
| **非会心ダメージ下限 / 上限** | 非会心時のダメージ範囲 |
| **Hit数** | そのカードの攻撃ヒット回数 |
| **会心率 (%)** | 会心が発生する確率（カード個別に設定可能） |
| **回避率 (%)** | 敵が回避する確率（カード個別に設定可能） |

### 操作

| 操作 | 方法 |
|---|---|
| カードの追加 | 「**+ ダメージ追加**」ボタン |
| カードの複製 | カード右上の「**📋**」ボタン（同じパラメータ・備考のカードが直後に追加されます） |
| カードの削除 | カード右上の「**✕**」ボタン（最低1枚は残ります） |
| カードの並べ替え | カード左上の「**⠿**」ハンドルをドラッグ＆ドロップ |
| メモの記入 | カード上部の備考欄に自由にテキストを入力 |

---

## 3. シミュレーション実行

1. 各カードのダメージ情報を入力
2. 目標ダメージを設定（任意）
3. 「**シミュレーション実行**」ボタンを押す

### 結果の見方

- **ヒストグラム**: 合計ダメージの分布（50万回のシミュレーション結果）
- **赤い破線**: 期待値（平均ダメージ）
- **緑の実線**: 目標ダメージ（設定時のみ）
- **通過確率**: 目標ダメージを超える確率がテキスト表示されます

---

## 4. 足切り機能


### 足切りラインの追加

1. 「**✂ 足切り追加**」ボタンを押す
2. 赤枠の足切りカードが追加される
3. ドラッグ＆ドロップで**足切りラインの位置**を変更 → ラインより上のカードが「上側」、下のカードが「下側」

### 足切り計算の実行

1. 足切りカード内の「**足切り計算**」ボタンを押す
2. 上側・下側それぞれの分布が計算される
3. 4つの連動スライダーが有効になる

### スライダーの意味

4つのスライダーは相互に連動しています。いずれか1つを動かすと、残り3つが自動で更新されます。

| スライダー | 意味 |
|---|---|
| **足切り値** | そこまでで達成するダメージ値 |
| **足切り通過確率** | そこまでで足切り値以上を出す確率 (%) |
| **残り必要ダメージ** | 目標ダメージ − 足切り値 = 下側グループで必要なダメージ |
| **残り通過確率** | それ以降でが残り必要ダメージ以上を出す確率 (%) |

### キャッシュについて

- カードのパラメータを変更すると、足切り計算のキャッシュが無効になります
- 再度「足切り計算」を押すと再計算されます
- パラメータ未変更なら、キャッシュが利用され高速に動作します
- ステータスに「再計算完了」または「キャッシュ利用」と表示されます


---

## 5. 操作のヒント

- **カードの並び順**は足切り計算に影響します。実際の攻撃順にカードを並べてください。
- **足切りラインは複数追加可能**です。
- **スライダー操作はリアルタイム**で反映されます
- 結果のヒストグラムはPlotlyで描画されており、**ズーム・パン・ホバー**で詳細を確認できます。
